# Оркестрация AI-помощника НТЦ Митра

Рекомендуемая схема:

1. Пользователь описывает дефект, отклонение или спорную ситуацию.
2. Система классифицирует ситуацию:
   - дефект СМР;
   - отклонение от проекта;
   - проблема ПИР;
   - исполнительная документация;
   - прочность бетона;
   - армирование;
   - геометрия;
   - фундаменты/сваи/основание;
   - трещины/прогибы/крен;
   - спор о необходимости усиления.
3. Система подбирает релевантные документы из локальной базы нормативки.
4. Агент 1 формирует черновой ответ и массив claims.
5. Агент 2 проверяет claims по источникам.
6. Финальный ответ пользователю формируется только после проверки.
7. Внутренний лог проверки сохраняется для аудита.

## Важное правило

Модель не должна подтверждать нормы по памяти. Нужен retrieval-слой по локальной базе нормативных документов или проверенным источникам.

## Рекомендуемая связка моделей

- Агент 1: модель, хорошо пишущая структурированные русскоязычные ответы.
- Агент 2: модель с жестким structured output и низкой температурой.
- При использовании DeepSeek и OpenAI: DeepSeek можно использовать для черновика, OpenAI - для верификации и финального structured output.

## Структурированный контракт между агентами

Агент 1 должен возвращать:

```json
{
  "draft_answer": "...",
  "claims": [
    {
      "doc": "...",
      "ref": "...",
      "claim": "...",
      "confidence": "requires_verification"
    }
  ],
  "needed_sources": ["..."],
  "user_questions": ["..."]
}
```

Агент 2 должен возвращать:

```json
{
  "final_answer": "...",
  "verification_log": {
    "checked_claims": [],
    "overall_risk": "low | medium | high",
    "requires_human_review": true
  }
}
```
